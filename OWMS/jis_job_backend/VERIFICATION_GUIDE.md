# 수동 검증 가이드 (상세 행 레이아웃)

## 서버 확인
개발 서버가 `localhost:3000`에서 실행 중입니다.

## Excel 파일 생성 방법

### 방법 1: 브라우저
아래 URL을 브라우저 주소창에 입력:
```
http://localhost:3000/reports/weekly-excel?userId=1&date=2026-02-10
```

Excel 파일이 자동 다운로드됩니다.

### 방법 2: PowerShell
```powershell
Invoke-WebRequest -Uri "http://localhost:3000/reports/weekly-excel?userId=1&date=2026-02-10" -OutFile "test_output_detailed.xlsx"
```

## 검증 항목

생성된 Excel 파일을 열어 다음을 확인하세요:

### ✅ 1. 업무제목 줄바꿈 처리
- 업무제목에 줄바꿈(`\n`)이 있으면 각 줄이 독립된 행을 점유하는지 확인
- 예: "프로젝트\nA\n개발" → 3개 행에 걸쳐 표시

### ✅ 2. 업무내용 줄바꿈 처리
- 업무내용도 줄바꿈 단위로 각 행을 점유하는지 확인
- 들여쓰기 3칸 적용 여부 확인 (`   내용`)

### ✅ 3. 일 간 공백
- 월요일 데이터 마지막 행과 화요일 첫 행 사이에 빈 행 1개 존재
- 화~수, 수~목, 목~금 사이에도 동일하게 빈 행 존재

### ✅ 4. 근무형태 표시 위치
- 각 일의 모든 업무 작성 후 마지막에 `<내근>`, `<외근>` 등 표시
- 공휴일/연차인 경우 `[공휴일: 설날]`, `[연차]` 형태로 표시

### ✅ 5. 요일 라벨
- 각 일의 **첫 번째 행**에만 `월(10)` 형식으로 요일 라벨 표시
- 두 번째 행 이후에는 요일 라벨 없음

### ✅ 6. 고정 행 레이아웃
- 템플릿의 41행(`중요정보사항`)이 원래 위치에 그대로 유지
- 행 확장이나 축소 없이 7-40행, 45-84행 범위 내에서만 데이터 기입

## 검증 스크립트 (선택사항)

Excel 파일 생성 후, 행 구조를 콘솔에서 확인:

```bash
node verify_detailed_layout.js
```

콘솔에 각 행의 요일 라벨과 내용이 표 형태로 출력됩니다.
